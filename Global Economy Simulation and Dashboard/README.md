# Ore Output Dashboard

Minimal R/Shiny skeleton for simulating stochastic ore output and the enchanted coin production pipeline supplied by the project owner. Broader economic modeling files are present as placeholders, but pricing, demand, trade costs, labor-market dynamics, shortages, inventories, and policy logic are intentionally not implemented.

## Run

```r
shiny::runApp(".", host = "127.0.0.1", port = 3838)
```

Then open <http://127.0.0.1:3838>.

Required R packages:

```r
install.packages(c("shiny", "yaml", "digest", "ggplot2"))
```

`ggplot2` is optional. Without it, the app falls back to base R plots.

## Shape Of The Project

- `app.R`: Shiny UI and reactive wiring only.
- `R/parameters.R`: YAML loading and dashboard-control overrides.
- `R/simulate_workers.R`: current stochastic mining worker process.
- `R/simulate_production.R`: converts worker output into tidy ore output.
- `R/money_supply.R`: converts ore output into minted, enchanted, reserved, and returned coin supply.
- `R/run_world_simulation.R`: parent `run_world_simulation()` and ore-output table assembly.
- `R/simulate_markets.R`: intentionally blank placeholder.
- `R/price_rules.R`: intentionally blank placeholder.
- `R/simulate_trade.R`: intentionally blank placeholder.
- `R/summarize_outputs.R`: ore-output summaries.
- `R/plotting_functions.R`: reusable plotting helpers.
- `scenarios/`: editable YAML scenario files.
- `data/`: optional saved run outputs.

## Current Scope

- Ores are `gold_ore`, `silver_ore`, and `copper_ore`.
- Mining production uses the shared `run_worker()`, `produce_output()`, and `produce_ore()` logic for every ore sector.
- Each worker success draws ore output between `output_min` and `output_max`.
- Dashboard output is the ore output generated by the mining process.
- Money supply output follows the provided minting/enchanting/reserve accounting.
- Leave `random_seed` blank for fresh stochastic production on each run, or set it for reproducible runs.
- Labor allocation remains a simple implicit setup parameter only. No labor-market dynamics are modeled.
- Pricing, demand, trade, trade costs, shortages, inventories, and market behavior are not modeled.
